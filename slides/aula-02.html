Organizando informações: Dados estruturados para gestão

Ao final desta aula, você será capaz de…

  • Distinguir listas, dicionários e DataFrames pela estrutura e pelo uso.
  • Construir uma tabela de gestão a partir de registros comerciais.
  • Selecionar colunas e filtrar linhas com critérios de negócio.
  • Calcular receita e justificar uma decisão com evidências.

Nosso roteiro de 3 horas

  • Conectar: do valor isolado ao registro estruturado.
  • Modelar: listas, dicionários, linhas e colunas.
  • Praticar: montar e explorar um DataFrame comercial.
  • Entregar: responder uma pergunta gerencial com dados.

Estrutura é contexto

O valor 189.90 sozinho diz pouco. Quando recebe o nome ticket_medio, pertence a uma filial e ocupa uma linha de período, ele se transforma em informação utilizável.

  • Variável: guarda um valor.
  • Lista: mantém uma sequência.
  • Dicionário: associa nomes a valores.
  • DataFrame: organiza muitos registros em tabela.

Listas: uma coleção ordenada

# Cada posição representa uma observação
vendas_semana = [120, 145, 138, 172, 160]

print(vendas_semana[0])  # primeira
print(vendas_semana[-1]) # última
print(sum(vendas_semana))

Índices começam em zero. Uma lista preserva ordem, mas não explica sozinha o significado de cada posição.

Dicionários: um registro com significado

filial = {
  "nome": "Paulista",
  "vendas": 320,
  "ticket_medio": 189.90
}

faturamento = filial["vendas"] * filial["ticket_medio"]

As chaves funcionam como rótulos: tornam a regra de negócio legível e reduzem ambiguidades.

Da ficha à tabela

Uma lista de dicionários representa várias fichas com o mesmo esquema. Cada dicionário vira uma linha; cada chave vira uma coluna.

registros = [
  {"filial": "Paulista", "vendas": 320, "ticket": 189.90},
  {"filial": "Pinheiros", "vendas": 280, "ticket": 214.50},
  {"filial": "Moema", "vendas": 410, "ticket": 176.00}
]

DataFrame: a tabela analítica

import pandas as pd

df = pd.DataFrame(registros)
df
  • Linhas são observações: filiais, clientes ou pedidos.
  • Colunas são atributos: vendas, região ou custo.
  • Índice identifica a posição de cada observação.

Inspecione antes de analisar

df.head()     # primeiras linhas
df.shape     # (linhas, colunas)
df.columns   # nomes das colunas
df.dtypes    # tipos de dados

Antes de calcular, confirme se a tabela tem a granularidade, as colunas e os tipos esperados.

Prática guiada: receita por filial

# Cria uma coluna calculada para cada linha
df["receita"] = df["vendas"] * df["ticket"]

# Seleciona uma visão executiva
painel = df[["filial", "receita"]]
painel.sort_values("receita", ascending=False)

Pergunta: qual filial lidera em receita — e por quê?

Filtrar é transformar regra em evidência

meta_receita = 60000

acima_da_meta = df[df["receita"] >= meta_receita]
acima_da_meta[["filial", "receita"]]

A expressão booleana é avaliada linha a linha. O resultado mantém somente os registros que satisfazem o critério.

Erros que custam decisões

  • Confundir posição e rótulo: índice zero não significa “filial 0”.
  • Misturar tipos: "R$ 1.000" é texto, não número.
  • Perder granularidade: misturar pedidos e totais mensais na mesma tabela.
  • Alterar sem conferir: valide shape, colunas e amostras após cada etapa.

Desafio: diagnóstico comercial

  • Calcule receita e atingiu_meta.
  • Ordene as filiais por receita e filtre as que bateram a meta.
  • Escreva uma recomendação de 2–3 frases citando dois indicadores.

Fechamento: do dado à decisão

  • Listas organizam sequências; dicionários descrevem registros.
  • DataFrames combinam registros em uma estrutura tabular.
  • Seleção, cálculo, ordenação e filtro respondem perguntas de gestão.
  • Uma análise só termina quando a evidência sustenta uma recomendação.
