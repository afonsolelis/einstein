Estratégia de estudo

Antes de executar uma célula, tente prever o formato do resultado: será um valor, uma Series ou um DataFrame? Essa previsão desenvolve leitura de código e reduz tentativas aleatórias.

Por que o mundo dos negócios usa dados tabulares?

Registros comerciais existem há séculos em livros-caixa, fichas e formulários. As planilhas eletrônicas tornaram o cálculo acessível, mas análises baseadas em cliques podem ser difíceis de repetir e auditar. O código acrescenta uma sequência explícita de transformações: outra pessoa pode executar as mesmas instruções sobre a mesma base e conferir o resultado.

O Pandas surgiu no ecossistema Python para facilitar a manipulação de dados rotulados e tabulares. Seu nome está associado a panel data, expressão usada para dados observados em múltiplas entidades e períodos. Hoje, seu DataFrame é uma ponte prática entre planilhas, bancos de dados e ferramentas analíticas.

A ferramenta, porém, não corrige uma estrutura conceitualmente errada. Antes de importar uma biblioteca, precisamos decidir o que será observado, com qual granularidade e quais atributos descrevem cada observação.

1. Do valor isolado ao dado estruturado

Imagine receber por mensagem apenas o número 189.90. Ele poderia ser um preço, uma distância ou uma taxa. Quando o associamos ao nome ticket_medio, à filial Paulista e ao mês de agosto, o número ganha contexto. Estruturar dados significa definir esse contexto de maneira consistente.

Em uma tabela de negócios, cada linha representa uma observação na mesma granularidade — por exemplo, uma filial em um mês. Cada coluna representa um atributo dessa observação, como região, vendas ou ticket médio. Essa regra parece simples, mas evita análises que misturam pedidos individuais, totais mensais e cadastros em uma mesma linha.

Observação

A entidade medida em uma linha: uma filial, um pedido ou um cliente.

Variável

Uma característica registrada em coluna: região, receita ou quantidade.

Granularidade

O nível de detalhe: “uma linha por filial por mês”, por exemplo.

Esquema

O contrato de nomes, significados e tipos esperados para as colunas.

2. Listas: sequências ordenadas

A lista é uma coleção ordenada e mutável. Ela é útil quando a posição tem significado ou quando queremos percorrer uma sequência de valores. Em Python, os índices começam em zero e índices negativos contam a partir do fim.

# Vendas de segunda a sexta-feira
vendas_semana = [120, 145, 138, 172, 160]

primeiro_dia = vendas_semana[0]
ultimo_dia = vendas_semana[-1]
total = sum(vendas_semana)
media = total / len(vendas_semana)

print(f"Total: {total} | Média: {media:.1f}")

Uma lista preserva a ordem, mas não descreve sozinha o que cada valor representa. Guardar ["Paulista", 320, 189.90] exige que todos memorizem o significado de cada posição — uma fonte comum de erros.

3. Dicionários: registros com significado

O dicionário associa chaves a valores. Ele funciona como uma ficha cadastral: em vez de pedir “a posição 1”, pedimos o valor da chave vendas. Isso aproxima o código da linguagem de negócio.

filial = {
    "nome": "Paulista",
    "regiao": "Centro",
    "vendas": 320,
    "ticket": 189.90
}

receita = filial["vendas"] * filial["ticket"]
filial["receita"] = receita
print(filial)

4. DataFrame: a tabela analítica

Uma lista de dicionários representa várias fichas com o mesmo esquema. O Pandas converte cada dicionário em uma linha e cada chave em uma coluna de um DataFrame. Essa estrutura combina a familiaridade de uma planilha com operações reproduzíveis em código.

import pandas as pd

registros = [
    {"filial": "Paulista", "regiao": "Centro", "vendas": 320, "ticket": 189.90},
    {"filial": "Pinheiros", "regiao": "Oeste", "vendas": 280, "ticket": 214.50},
    {"filial": "Moema", "regiao": "Sul", "vendas": 410, "ticket": 176.00}
]

df = pd.DataFrame(registros)
df

5. Inspeção: conheça a base antes do cálculo

Uma análise responsável começa pela inspeção. head() mostra uma amostra, shape informa dimensões, columns lista os nomes e dtypes revela como o Pandas interpretou cada coluna.

print(df.head())
print("Dimensões:", df.shape)
print("Colunas:", df.columns.tolist())
print(df.dtypes)

Granularidade antes da ferramenta

Defina o que uma linha representa antes de montar a tabela. Não misture, por exemplo, uma linha por pedido com outra linha que já seja o total do mês. Verifique também se valores monetários são números: "R$ 1.000" é texto e não pode ser somado corretamente.

Tipos de dados e validação do esquema

O tipo determina quais operações são válidas. Números podem ser somados; textos podem ser concatenados; booleanos representam condições verdadeiras ou falsas. No Pandas, os tipos mais comuns neste exercício são object ou string para texto, int64 para inteiros, float64 para decimais e bool para indicadores lógicos.

# Conferências simples antes da análise
print(df.dtypes)
print("Nulos por coluna:")
print(df.isna().sum())

# Regras mínimas para este conjunto
assert df["filial"].notna().all(), "Há filial sem nome"
assert (df["vendas"] >= 0).all(), "Vendas não podem ser negativas"
assert (df["ticket"] >= 0).all(), "Ticket não pode ser negativo"

As instruções assert funcionam como alarmes: se uma regra for falsa, a execução para com uma mensagem clara. Elas não substituem uma política completa de qualidade, mas tornam premissas importantes visíveis no notebook.

Seleção e indexação: coluna, linha e célula

Um DataFrame tem rótulos de coluna e um índice para as linhas. O índice padrão começa em zero, mas não deve ser confundido com uma identificação de negócio. Para uma operação legível, prefira selecionar explicitamente linhas e colunas.

Formas frequentes de selecionar dados
ExpressãoResultadoQuando usar
df["vendas"]SeriesUma coluna
df[["filial", "vendas"]]DataFrameVárias colunas
df.loc[condicao, colunas]Series ou DataFrameRótulos e filtros booleanos
df.iloc[0:2, 0:3]DataFramePosições numéricas
# Linhas com pelo menos 300 vendas; somente duas colunas
alto_volume = df.loc[
    df["vendas"] >= 300,
    ["filial", "vendas"]
]

# Duas primeiras linhas e três primeiras colunas, por posição
amostra = df.iloc[0:2, 0:3]

loc e iloc não são sinônimos

loc trabalha com rótulos e inclui o limite final de um intervalo rotulado. iloc trabalha com posições e exclui o limite final, seguindo o padrão de fatias do Python.

6. Prática guiada: painel de filiais

Vamos criar um indicador por linha, selecionar uma visão executiva e ordenar o resultado. A atribuição abaixo é vetorizada: a multiplicação acontece para todas as linhas sem copiar fórmulas.

# Receita de cada filial
df["receita"] = df["vendas"] * df["ticket"]

# Seleção de colunas e ordenação decrescente
painel = df[["filial", "regiao", "receita"]]
ranking = painel.sort_values("receita", ascending=False)
print(ranking)

Em seguida, traduza uma meta gerencial em uma condição booleana. O Pandas avalia a condição em cada linha e mantém apenas as linhas verdadeiras.

meta_receita = 60000
df["atingiu_meta"] = df["receita"] >= meta_receita

filiais_na_meta = df.loc[
    df["atingiu_meta"],
    ["filial", "receita"]
]
print(filiais_na_meta)

7. Seleção, filtro e ordenação sem confusão

  • df["receita"] seleciona uma coluna e retorna uma Series.
  • df[["filial", "receita"]] seleciona várias colunas e retorna um DataFrame.
  • df.loc[condicao, colunas] escolhe linhas e colunas por rótulo.
  • sort_values() muda a ordem da visualização, não o significado das linhas.

Evite encadear operações longas enquanto aprende. Nomear resultados intermediários, como painel e ranking, facilita a conferência e a explicação do raciocínio.

Análise consolidada: do cálculo à recomendação

Um indicador isolado raramente conta toda a história. Receita combina volume e ticket; por isso, uma filial pode liderar vendendo muitas unidades com ticket menor, enquanto outra pode compensar menor volume com ticket maior. Compare os componentes antes de recomendar uma ação.

# Indicadores gerais do pequeno painel
receita_total = df["receita"].sum()
ticket_medio_simples = df["ticket"].mean()
lider = df.sort_values("receita", ascending=False).iloc[0]

print(f"Receita total: R$ {receita_total:,.2f}")
print(f"Ticket médio entre filiais: R$ {ticket_medio_simples:,.2f}")
print(f"Líder: {lider['filial']} — R$ {lider['receita']:,.2f}")

Observe a precisão da linguagem: df["ticket"].mean() é a média simples dos tickets das filiais, não necessariamente o ticket médio de todas as transações. Para este último, seria necessário calcular uma média ponderada:

ticket_ponderado = (
    (df["ticket"] * df["vendas"]).sum()
    / df["vendas"].sum()
)
print(f"Ticket ponderado: R$ {ticket_ponderado:.2f}")

Boa recomendação gerencial

Use a estrutura “evidência → interpretação → ação”. Exemplo: “Moema lidera a receita por combinar o maior volume com ticket competitivo; vale investigar suas práticas comerciais antes de replicá-las nas demais filiais.”

8. Desafio: diagnóstico comercial

Crie um notebook chamado Aula_02_SeuNome.ipynb. Monte um DataFrame com cinco filiais e as colunas filial, regiao, vendas, ticket e meta. Você pode usar valores fictícios coerentes.

  1. Calcule receita = vendas * ticket.
  2. Crie atingiu_meta comparando receita e meta de cada linha.
  3. Ordene as filiais da maior para a menor receita.
  4. Filtre as filiais que atingiram a meta.
  5. Escreva uma recomendação de duas a três frases citando pelo menos dois indicadores calculados.

Critério de entrega: todas as células devem executar de cima para baixo sem erro; o DataFrame final deve conter sete colunas; o ranking, o filtro e a recomendação devem estar visíveis.

Resolução comentada do desafio

Tente concluir o desafio antes de abrir esta seção. A solução abaixo é uma referência possível, não a única. Os dados são fictícios e foram escolhidos para produzir casos acima e abaixo da meta.

import pandas as pd

dados = [
    {"filial": "Paulista", "regiao": "Centro", "vendas": 320, "ticket": 189.90, "meta": 60000},
    {"filial": "Pinheiros", "regiao": "Oeste", "vendas": 280, "ticket": 214.50, "meta": 62000},
    {"filial": "Moema", "regiao": "Sul", "vendas": 410, "ticket": 176.00, "meta": 70000},
    {"filial": "Santana", "regiao": "Norte", "vendas": 250, "ticket": 198.00, "meta": 52000},
    {"filial": "Tatuapé", "regiao": "Leste", "vendas": 360, "ticket": 181.50, "meta": 64000}
]

df = pd.DataFrame(dados)
df["receita"] = df["vendas"] * df["ticket"]
df["atingiu_meta"] = df["receita"] >= df["meta"]

A comparação usa df["meta"], e não uma constante, porque cada filial tem sua própria meta. O resultado contém exatamente as sete colunas exigidas.

ranking = df.sort_values("receita", ascending=False)
na_meta = df.loc[
    df["atingiu_meta"],
    ["filial", "receita", "meta"]
]

print(ranking[["filial", "vendas", "ticket", "receita"]])
print(na_meta)

Leitura possível: Moema lidera com receita de R$ 72.160,00, apoiada pelo maior volume de vendas, e supera sua meta de R$ 70.000,00. Paulista e Tatuapé também superam suas metas; uma ação coerente seria comparar as práticas das três filiais e investigar por que Pinheiros, apesar do maior ticket, não alcança a meta devido ao volume menor.

Glossário essencial

DataFrame
Estrutura tabular bidimensional com linhas e colunas rotuladas.
Series
Estrutura unidimensional; normalmente representa uma coluna.
Índice
Rótulo usado pelo Pandas para identificar linhas; não é automaticamente uma chave de negócio.
Máscara booleana
Sequência de valores verdadeiros e falsos usada para filtrar linhas.
Vetorização
Operação aplicada a uma coluna inteira sem escrever um laço explícito linha a linha.
Granularidade
Nível de detalhe representado por cada linha.
